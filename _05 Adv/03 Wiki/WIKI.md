# Wiki

A RAG system that aims to encode your relationships to the sources. By creating an agent that analyzes your vault for patterns it can be scheduled and instructed to maintain its own wiki. Becomes a graph of linked notes with a set of relationships determined by your use of them and how they relate to you. Not a straightforward setup but is a large improvement from pure semantic chunking.

## Wiki vs. Notes vs. Raw Sources

| Layer             | Purpose                                                        | Mutability                            |
| ----------------- | -------------------------------------------------------------- | ------------------------------------- |
| `01 Raw Sources/` | Captured inputs — articles, PDFs, transcripts, clippings       | Immutable — keep as ingested          |
| `04 Notes/`       | Working thoughts, drafts, project notes, in-progress synthesis | Changes constantly                    |
| `03 Wiki/`        | Distilled, cross-referenced reference notes you return to      | Stable, curated, updated deliberately |

Think of it as the "published" shelf. A note collects your interpretations and reasoning behind sources you add. Documentation managed by an agent to allow for knowledge graph and real connections of context. Can be read and edited but effective as LLM managed and referenced.

## What belongs here


- Glossaries, cheatsheets, opinionated summaries of a topic, **reasoning**
- Wiki-style entries on tools, people, places, systems that you reference repeatedly
- Im not sure yet, the wiki should be agent managed or augmented.
- There is a lot of people researching this still and other good sources available.
